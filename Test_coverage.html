Software Test Coverage

This case study shows how I use automated coverage thresholds, component tests, and smoke tests to keep frontend changes safe to ship in a production TypeScript codebase.

How I enforce it

  • Unit tests for utilities, selectors, and hooks using Jest and ts-jest.
  • Component tests for pagination, filters, and debounced search with React Testing Library.
  • MSW-backed API mocks for slow responses, malformed data, and failure states.
  • GitHub Actions checks that block merges when coverage drops below agreed thresholds.
  • Nightly Playwright smoke tests against preview deployments.
